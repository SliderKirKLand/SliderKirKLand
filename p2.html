Excercise 3

In this exercise, I worked with both parent and child functions to create a dynamic and interactive visual environment. The parent functions in my code include drawGhost() and moveGhost(), which control the overall behavior of each ghost. These parent functions call child functions such as ghost(), body(), whiteEye(), blueEye(), and leg(). The child functions are responsible for drawing different parts of the ghost, while the parent functions manage positioning, movement, scaling, and rotation of the ghosts. This structure helps modularize the code, making it easier to manage different behaviors and visual effects for each ghost.

I feel that I met the exercise requirements well, incorporating elements like vector-based movement, boundary collision, and trail effects. I also implemented scaling and rotation, adding variety to the size and movement of the ghosts. The trails now change color randomly, creating a vibrant, ever-changing visual dynamic. Overall, I believe this was a successful implementation of the exercise, and it allowed me to experiment with a range of programming concepts.

Excercise 4

In this project, I created a fun, interactive digital scene where ants move around pieces of cheese. The main functions in the code manage how the ants behave and move around, with randomized speeds and paths to mimic real ant movements. I also set it up so viewers can drag the cheese pieces, which changes the ants’ routes, making the environment more engaging and dynamic.

The design focuses on simple shapes—triangles for the cheese and ellipses for the ants—keeping the visuals easy to interact with. While the design is straightforward, it still offers depth through its background details. Overall, this project combines interaction and movement, creating a playful digital experience that’s both fun and visually interesting.

Project 2

In this project use custom objects that move and change across the screen depending on the mode. Each object has its own movement and shape, and their behavior shifts with each mode. In Mode 0, they fall like snowflakes. Mode 1 makes them follow sine and cosine wave patterns, while Mode 2 makes them move in circles around the center before launching them outward in random directions. These changes keep the visuals interesting and let viewers see how each mode affects the objects.

There’s also a recording and playback option, so viewers can capture and rewatch their interactions, making each session feel unique. The colors of the objects change based on their position, adding more depth as they move across the screen. With easy controls to switch modes and record what happens, this piece combines art and technology in a simple but engaging way, creating a dynamic experience that’s always changing.
